# Product UI/UX System for Guided Operator Workflows

Design every surface for an operator who wants to get from raw input to a real outcome quickly, clearly, and confidently.

This skill is especially relevant for MenuAI, where the user is a restaurant operator trying to move from raw menu content to a guest-ready result.

## Core Rule
The product must feel like a guided AI operating system, not a collection of tools, not a generic SaaS dashboard, and not a chatbot workflow.

## Always optimize for
- clarity
- actionability
- trust
- operator confidence
- visible progress
- visible outcome
- cross-surface coherence
- commercial honesty

## Operator-first principles
- One clear next step beats many equal options
- Awareness must connect to action
- Guidance must be embedded where the user is already working
- AI may prepare, but the operator approves
- Before/after value must be felt, not hidden
- Empty states must start a workflow, not just describe emptiness
- Copy must explain what happened, why it matters, and what to do next
- If something is unavailable, paid, early access, or not yet active, say so clearly and honestly

## Surface roles
- Dashboard = current state + next best step
- Workspace / Menu Builder = main working environment
- AI Studio = focused enhancement work
- Preview = outcome validation
- Settings = configuration, not workflow

Do not blur these roles without strong product reason.

## Hierarchy rules
- One primary action
- Secondary actions must look secondary
- Tertiary actions must not compete with the main path
- Do not give equal visual weight to unequal actions

## Reject these patterns
- decorative UI without product value
- passive stats with no action path
- dead ends
- duplicate guidance across surfaces
- generic enterprise dashboard behavior
- unexplained badges, numbers, or labels
- fake upgrade or payment implications
- placeholder controls
- feature drift
- chatbot-like behavior where deterministic guidance is better

## MenuAI-specific standards
- The product must support 3 clear start paths:
  1. import existing menu
  2. create manually
  3. scan/upload dish photo
- The user must reach a first visible result quickly
- The user must understand what matters next
- The user must be guided toward improvement, review, preview, and QR-ready outcome
- The workspace is the product center of gravity
- AI guidance should feel like a smart operator buddy, not a separate destination
- Commercial framing must always be truthful, especially around plans, credits, paid features, and early access

## Evaluate every screen by asking
- Where am I?
- What matters now?
- What should I do next?
- What outcome will I get?
- Is this helping the operator act, or just showing information?

## Final Rule
If a change makes the UI prettier but less clear, reject it.
If a change makes the product more actionable, more trustworthy, and more guided, prefer it.
